<div class="space-y-2"><div class="flex max-w-full flex-col gap-4 grow"><div data-message-author-role="assistant" data-message-id="7ee3f053-57c0-4e08-9321-f9a97b4196e8" dir="auto" class="min-h-8 text-message relative flex w-full flex-col items-end gap-2 text-start break-words whitespace-normal outline-none keyboard-focused:focus-ring [.text-message+&amp;]:mt-1"><div class="flex w-full flex-col gap-1 empty:hidden"><div class="markdown prose dark:prose-invert w-full wrap-break-word dark markdown-new-styling"><h1 data-start="0" data-end="16">RusticalandOPS</h1>
<p data-start="18" data-end="404"><strong data-start="18" data-end="36">RusticalandOPS</strong> is a modular operations stack for running and administering Rust dedicated game servers on a self‑hosted Linux machine.  It combines deterministic lifecycle management, an HTTP API, an AI‑assisted admin agent and a Steam chat transport adapter.  The project aims to provide a local‑first alternative to hosted control panels while remaining extensible and scriptable.</p>
<h2 data-start="406" data-end="417">Features</h2>
<ul data-start="419" data-end="2033">
<li data-start="419" data-end="655">
<p data-start="421" data-end="655"><strong data-start="421" data-end="454">Deterministic server control:</strong> A Bash script (<code data-start="470" data-end="482">rustmgr.sh</code>) manages starting, stopping, updating, wiping and querying multiple Rust servers.  It uses <code data-start="574" data-end="580">tmux</code> to run servers in the background and parses JSON configuration per server.</p>
</li>
<li data-start="656" data-end="839">
<p data-start="658" data-end="839"><strong data-start="658" data-end="671">REST API:</strong> An ASP.NET&nbsp;8 service (<code data-start="694" data-end="706">rustmgrapi</code>) exposes lifecycle, log, player, RCON, plugin and network operations over HTTP.  The built‑in <code data-start="801" data-end="806">/ui</code> route serves a simple dashboard.</p>
</li>
<li data-start="840" data-end="1270">
<p data-start="842" data-end="1270"><strong data-start="842" data-end="858">Admin agent:</strong> A long‑running C# process (<code data-start="886" data-end="900">RustOpsAgent</code>) watches local inbox folders for admin requests, classifies intent using heuristics or optional LLM models, routes work to tool handlers, writes semantic memories to a local SQLite store and produces replies.  Tool handlers currently include server control, status, player lookup, RCON, log inspection, plugin updates, network info, file editing and chat summarisation.</p>
</li>
<li data-start="1271" data-end="1523">
<p data-start="1273" data-end="1523"><strong data-start="1273" data-end="1300">Steam chat integration:</strong> A SteamKit2‐based bot (<code data-start="1324" data-end="1337">OpsSteamBot</code>) relays chat messages from administrators to the agent via the file inbox, and sends replies back through Steam chat.  It also forwards explicit <code data-start="1483" data-end="1492">approve</code>/<code data-start="1493" data-end="1501">reject</code>/<code data-start="1502" data-end="1512">feedback</code> decisions.</p>
</li>
<li data-start="1524" data-end="1775">
<p data-start="1526" data-end="1775"><strong data-start="1526" data-end="1549">Local‑first design:</strong> All components communicate over local files and HTTP on <code data-start="1606" data-end="1617">localhost</code>.  External services such as LLM endpoints and embedding providers are optional; the system continues to function deterministically when they are unavailable.</p>
</li>
<li data-start="1776" data-end="2033">
<p data-start="1778" data-end="2033"><strong data-start="1778" data-end="1798">Semantic memory:</strong> The agent stores facts, procedures, failures, fixes and user instructions in a local SQLite database.  It performs vector search using embeddings fetched from an OpenAI‑compatible endpoint and can migrate legacy state from JSON files.</p>
</li>
</ul>

<h2 data-start="2650" data-end="2668">Getting Started</h2>
<h3 data-start="2670" data-end="2687">Prerequisites</h3>
<ul data-start="2689" data-end="3039">
<li data-start="2689" data-end="2755">
